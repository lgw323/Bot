# 봇 건강 검진(Test) 가이드 🩺

안녕하세요! 이 문서는 프로그래밍 지식이 없는 관리자님들도 디스코드 봇이 아프지 않고 정상적으로 작동하는지 쉽게 확인해 볼 수 있도록 쓰여진 가이드입니다. 

---

## 💡 1. 시스템 안정성 개요: 테스트(건강 검진)가 왜 필요할까요?
로봇청소기가 벽에 부딪혔을 때 어떻게 피해야 할지 매번 상황을 만들어 알려주는 것을 **'테스트'**라고 합니다.
디스코드 봇도 마찬가지입니다. 유저들이 수백, 수천 명 모여 동시에 명령어를 치거나 장난을 칠 때 봇이 다운(기절)되지 않고 튼튼하게 버티려면 미리 가상의 시나리오로 예방 접종을 맞춰야 합니다.

우리가 작성한 이 **'건강 검진 코드(Test Code)'**는 실제로 봇을 켜보지 않아도, 컴퓨터가 눈 깜짝할 새에 40여 가지가 넘는 상황극을 시뮬레이션하여 봇의 상태를 100점 만점으로 평가해주는 안전망입니다!

## 📋 2. 기능별 검사 항목 (어떤 상황을 방어하나요?)

다음은 우리의 봇이 어떤 가상의 훈련을 거치는지 일상적인 언어로 정리한 표입니다.

| 기능 이름 | 검사 항목 (이런 상황을 방어합니다!) |
| :--- | :--- |
| **🗄️ 데이터베이스 (DB)** | **동시에 몰려도 줄서기:** 여러 명이 동시에 XP(경험치)를 얻거나 노래 추천을 눌러도, 데이터베이스가 고장 나거나 기록이 엉키지 않게 순서대로 처리되는지 검사합니다. <br> **안전한 백업 및 보안:** 갑자기 정전이 되어 데이터가 통째로 날아가도 백업본(SQL)으로 정상 복구되는지 확인하며, 이 백업본이 인터넷에 유출되어도 유저 개인정보가 '외계어'로 완벽히 암호화 처리되는지 깐깐하게 검사합니다. |
| **📈 레벨링 (경험치)** | **정확한 성적표:** 채팅 "안녕하세요"를 쳤을 때 글자 수(자음, 모음)에 정확히 비례해서 경험치를 받는지 검사합니다. |
| **📝 로깅 (알람 창구)** | **긴급 호출:** 봇 내부에서 치명적인 에러(에러 레벨)가 나면, 봇이 혼자 끙끙 앓지 않고 관리자 전용 디스코드 알람 채널로 즉시 삐용삐용 문자를 보내는지 검사합니다. <br> **말 줄임표:** 에러 내용이 너무 길어서 디스코드 메신저 제한을 넘기면, 똑똑하게 말을 잘라서라도 무조건 보내는지 확인합니다. |
| **🎶 음악 시스템 (Music)** | **동시 접속 방어:** 두 명의 유저가 동시에 노래를 틀어달라고 하거나 스킵(넘기기) 버튼을 파바박 눌렀을 때 봇이 뻗지 않는지 검사합니다. <br> **유튜브 링크 확인:** 이상한 주소가 아닌 진짜 유튜브 주소만 골라내서 재생하는지 확인합니다. |
| **🤖 대화 요약 (AI)** | **메모리 정리:** 수백 개의 채팅이 쌓여 요약을 부탁했을 때, 봇의 머리(메모리)가 터지지 않도록 오래된 대화는 스스로 잊어버리고 최근 것만 잘 추려서 AI에게 전달하는지 검사합니다. |
| **🛠️ 시스템 무결성** | **부품 조립 불량 검사:** 심장(main_bot)부터 팔다리(cogs)까지 코드 여러 개 중 하나라도 이름이 틀리거나 조립이 덜 된 부분이 있으면 즉각 경고를 주는지 검사합니다. |

## 🚀 3. 일반인용 실행 가이드 (직접 검사해보기)

관리자님이 봇 코드를 조금이라도 수정했거나 업데이트를 받았다면, 봇을 실제 서버에 가동하기 전에 **반드시 건강 검진을 실행**해보세요. 아주 쉽습니다!

### 🖥️ 실행 방법
1. 봇 폴더가 열려 있는 검은색 창(터미널)을 클릭합니다.
2. 아래 명령어를 그대로 복사해서 붙여넣고 키보드 `Enter` 키를 누르세요.
   ```bash
   pytest tests/
   ```

### ✅ 검사 통과 시 화면 (정상)
약 1~2초 후, 화면 아래쪽에 **초록색 글씨**로 `passed`가 뜨면 봇이 아주 건강하다는 뜻입니다.  
> **`=================== 44 passed in 1.78s ===================`**

### ❌ 검사 실패 시 화면 (비정상)
만약 화면에 **빨간색 글씨**로 `FAILED` 가 여러 개 뜬다면, 방금 수정한 코드에 문제가 있어 봇이 기절할 수도 있다는 뜻입니다. 
> **`=================== 2 failed, 42 passed in 2.10s ===================`**
이때는 봇을 켜지 말고 에러 내용을 스크린샷으로 찍어 개발자에게 전달하시면 됩니다!
