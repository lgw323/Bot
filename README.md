# DiscordBot (다기능 디스코드 봇)

이 프로젝트는 파이썬(discord.py)으로 작성된 다기능 디스코드 봇입니다. 
주요 기능으로 **노래 재생**, **대화 요약(AI)**, **경험치/레벨링 시스템**, 그리고 **전역 로깅 시스템**을 포함하고 있습니다.

---

## 📂 프로젝트 구조 및 파일 명세 (기능 명세서)

### 1. 코어 및 실행 파일 (Root Directory)
*   `main_bot.py`: **봇의 심장이자 진입점(Entry Point).** 
    *   봇 객체를 생성하고, 각종 `Cogs`(기능 모듈)를 로드하며 디스코드 서버에 연결합니다.
    *   앱 실행 시 초기 명령어 동기화 및 데이터베이스 마이그레이션(`database_manager.py` 호출)을 수행합니다.
*   `database_manager.py`: **데이터베이스 코어 매니저.**
    *   봇 전체에서 사용하는 SQLite 데이터베이스(`data/bot_database.db`)의 스키마를 정의하고 생성합니다.
    *   기존 JSON 파일(`music_settings.json`, `favorites.json`)에서 SQLite DB로 데이터를 이관하는 마이그레이션 스크립트를 포함합니다.
    *   **설계 의도**: 개별 Cog 안에 종속되지 않고 최상단에 위치하여, 여러 기능(음악, 레벨링 등)이 DB에 동시 접근할 때 충돌을 맵핑하고 의존성을 최소화합니다.
*   `requirements.txt`: 파이썬 패키지 의존성 목록.

### 2. 스크립트 및 문서 (Scripts & Docs)
*   `docs/server_setup.md`: 라즈베리파이나 서버 구축 시 필요한 기본 환경 설정 가이드.
*   `scripts/auto_backup.sh`, `scripts/auto_update.sh`: 서버 자동 백업 및 깃허브 최신화(자동 업데이트)를 위한 리눅스 쉘 스크립트.

### 2. 기능 모듈 (Cogs Directory)
디스코드 봇의 각 세부 기능들은 객체지향적 책임을 분리하기 위해 `cogs/` 폴더 내에 기능별로 흩어져 있습니다.

#### 🎶 Music (음악 시스템) - `cogs/music/`
*   `music_agent.py`: **음악 시스템의 메인 프록시(Controller).** 
    *   유저의 디스코드 명령어(`/재생`, `/스킵` 등)를 리슨(Listen)하고, `music_core`나 UI로 요청을 분배합니다.
    *   봇이 음성 채널에 참여하거나 나갈 때 TTS(텍스트 음성 변환) 알림을 방송하는 로직을 포함합니다.
*   `music_core.py`: **실제 재생기 (Player).**
    *   YoutubeDL(yt-dlp)과 FFmpeg를 사용해 스트리밍 오디오를 디스코드 음성 클라이언트로 쏩니다. 대기열(Queue) 관리, 재생, 곡 넘김 등 핵심 재생/일시정지 상태 관리를 수행합니다.
*   `music_ui.py`: **유저 인터페이스 (View).**
    *   디스코드의 버튼, 셀렉트 메뉴 등 상호작용 가능한 UI 위젯들(즐겨찾기 뷰, 검색 결과 뷰 등)을 렌더링하고 유저의 클릭에 반응합니다.
*   `music_utils.py`: **음악 유틸리티.** 
    *   상수(Constants), 정규표현식, 즐겨찾기 목록 불러오기 및 DB 함수형 래퍼(Wrapper), 구조체(Song 객체) 등이 모여있습니다.

#### 🧠 Summary (대화 요약 시스템) - `cogs/summary/`
*   `summary_listeners.py`: **디스코드 이벤트 리스너.**
    *   지정된 채널의 메시지를 실시간으로 모니터링하여 임시 메모리(Deque)에 적재합니다. 
    *   UI 버튼이나 명령어를 통해 호출되면 쌓인 로그를 모아 `summarizer_agent.py`에게 전달합니다.
*   `summarizer_agent.py`: **AI 요약 엔진.**
    *   Google Gemini API(혹은 OpenAI)와 통신하여 넘겨받은 채팅 로그들을 프롬프트에 담아 분석 및 구조화(핵심 요지, 세부 내용)된 JSON/텍스트 형식으로 변환하여 반환합니다.

#### 📈 Leveling (경험치 시스템) - `cogs/leveling/`
*   `leveling_core.py`: **채팅 및 음성 채널 레벨 관제 (새로 추가됨).**
    *   지정된 채널에서의 모든 채팅에 대해 **한글 자/모음 개수를 분해하여 공평하게 XP를 부여**합니다.
    *   음성 채널 진입/이탈 시간을 추적해 분 단위 체류 경험치를 부여합니다.
    *   경험치가 목표치에 달성하면 디스코드의 역할(예: `Lv.10 수다쟁이`)을 자동으로 교체 및 부여하며, 유저 조회 명령어(`/내정보`, `/랭킹`)를 제공합니다.

#### 📝 Logging (로깅 시스템) - `cogs/logging/`
*   `log_agent.py`: **전역 로깅 시스템.**
    *   파이썬의 내장 `logging` 시스템을 하이재킹하여 콘솔과 로컬 파일(`data/logs/system.log`)에 로그를 남깁니다.
    *   특히 에러 레벨(Error) 이상의 치명적인 오류 발생 시, 봇이 지정된 디스코드 채널로 백트레이스 정보가 담긴 알람 메시지를 즉각 전송합니다.

#### ⚙️ Common 
*   `application_commands.py`: 전역 슬래시 명령어 유틸(기본적인 핑 테스트 등 소규모 명령어 모음).

### 3. 데이터 컨테이너 (Data Directory)
이 폴더는 깃허브(버전 관리)에서 주로 제외되며, 로컬 라즈베리파이(서버) 환경에서 직접 유지/관리되는 실제 데이터베이스 및 백업 폴더입니다.
*   `bot_database.db`: 봇의 사용자 경험치 레벨, 음악 볼륨 세팅, 개인 즐겨찾기 노래 등을 영구 보관하는 SQLite3 데이터베이스 파일.
*   `logs/`: 날짜별로 순환(Rotation) 생성되는 `system.log` 파일들이 보관됩니다.
*   `*.json.bak`: 기존 아키텍처에서 사용하던 설정 파일들이 DB 통합 마이그레이션 후 백업된 파일들.
