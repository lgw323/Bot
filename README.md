# DiscordBot (다기능 디스코드 봇)

이 프로젝트는 파이썬(discord.py)으로 작성된 다기능 디스코드 봇입니다. 
주요 기능으로 **노래 재생**, **대화 요약(AI)**, **경험치/레벨링 시스템**, 그리고 **전역 로깅 시스템**을 포함하고 있습니다.
또한 `pytest` 기반의 탄탄한 테스트 코드가 구축되어 있어 안정성이 보장되는 봇 플랫폼입니다.

---

## 📂 프로젝트 구조 및 핵심 기능 명세

### 1. 코어 시스템 (Root Directory)
*   `main_bot.py`: **봇의 심장이자 진입점(Entry Point).** 
    *   봇 객체를 생성하고, 각종 `Cogs`(기능 모듈)를 로드하며 디스코드 서버에 연결합니다.
    *   앱 실행 시 초기 명령어 동기화 및 데이터베이스 스키마 초기화(`database_manager.py` 호출)를 수행합니다.
*   `database_manager.py`: **데이터베이스 코어 매니저.**
    *   봇 전체에서 사용하는 SQLite 데이터베이스(`data/bot_database.db`)의 스키마를 정의하고 관리합니다.
    *   **설계 의도**: 개별 모듈 안에 DB가 종속되지 않고 최상단에 위치하여, 여러 기능(음악, 레벨링 등)이 동시 접근할 때 충돌을 방지(Concurrency Control)하고 의존성을 최소화합니다.
    *   **보안/백업 설계**: 깃허브 `db-backup` 브랜치로 SQL 덤프 파일(`.sql`) 단일 커밋 강제 덮어쓰기 백업을 수행하여 용량 팽창을 방지합니다. 덤프 시 유저의 민감한 기록은 Fernet 암호화 처리되며, 로컬 DB가 파괴되었을 경우 봇 부팅 시 깃허브에서 백업본을 자동 다운로드 및 해독 복구하는 완전 자동화된 재난 복구(DR) 계층이 적용되어 있습니다.
*   `requirements.txt` / `requirements-dev.txt`: 파이썬 패키지 의존성 목록 (실행 환경 및 개발/테스트 환경 분리).

### 2. 스크립트 및 문서 (Scripts & Docs)
*   `docs/`: 라즈베리파이 서버 구축, 테스트 가이드, 개발 정책(Project Context) 등 봇 운영 및 유지보수에 필요한 문서들이 포함되어 있습니다.
*   `scripts/`: 서버 자동 백업 및 깃허브 최신화(자동 업데이트)를 위한 리눅스 쉘 스크립트 모음.

### 3. 기능 모듈 (Cogs Directory)
디스코드 봇의 각 세부 기능들은 객체지향적 책임을 분리하기 위해 `cogs/` 하위에 기능별로 흩어져 있습니다.

#### 🎶 Music (음악 시스템) - `cogs/music/`
*   `music_agent.py`: **음악 시스템의 메인 프록시(Controller).** 
    *   유저의 디스코드 명령어(`/재생`, `/스킵` 등)를 받아 `music_core`나 `music_ui`로 요청을 분배합니다.
*   `music_core.py`: **실제 재생기 (Player).**
    *   YoutubeDL(yt-dlp)과 FFmpeg를 사용해 스트리밍 오디오를 디스코드 음성 클라이언트로 송출합니다. 대기열(Queue) 로직과 코어 재생/일시정지 상태를 전담합니다.
*   `music_ui.py`: **유저 인터페이스 (View).**
    *   디스코드의 버튼, 셀렉트 메뉴 등 상호작용 가능한 UI 위젯을 렌더링하고 유저 클릭에 반응합니다.
*   `music_utils.py`: **음악 유틸리티.** 
    *   비동기 DB 조회, 상태 캐싱(`JSON State Saver` - 봇 재시작시 재생 구간 및 대기열 임시 보존/복구 모델) 등의 I/O 로직을 총괄합니다.

#### 🧠 Summary (대화 요약 시스템) - `cogs/summary/`
*   `summary_listeners.py`: 지정된 채널의 메시지를 실시간으로 모니터링하여 임시 메모리에 적재합니다.
*   `summarizer_agent.py`: **AI 요약 엔진.** API(Google Gemini 등)와 통신하여 채팅 로그들을 요약(핵심 요지, 세부 내용)하고, 알기 쉬운 구조로 변환하여 채널에 응답합니다.

#### 📈 Leveling (경험치 시스템) - `cogs/leveling/`
*   `leveling_core.py`: **채팅 및 음성 채널 경험치 관리자.**
    *   지정된 채널의 메시지에서 **한글 자/모음 개수를 분해하여 공평하게 XP를 부여**합니다.
    *   음성 채널 진입/이탈 시간을 추적해 분 단위 체류 경험치를 부여합니다.
    *   경험치가 목표치에 달성하면 레벨이 오르며, 유저 조회 명령어(`/내정보`, `/랭킹`)를 제공합니다.

#### 📝 Logging (전역 로깅) - `cogs/logging/`
*   `log_agent.py`: 파이썬 내장 기능을 하이재킹하여 콘솔 및 로컬 파일(`data/logs/system.log`)에 로그를 남기고, 치명적인 오류 발생 시 지정된 디스코드 알람 채널로 에러 내용을 즉시 전송합니다.
    *   서버 로그 채널을 통해 수동으로 봇을 최신화하고 재구동(Restart)할 수 있는 **마스터 관리자 전용 제어 패널 UI**를 함께 제공합니다.

### 4. 테스트 시스템 (tests/)
*   **단위/통합 테스트 완비**: 봇이 멈추거나 꼬이는 상황을 방지하기 위해 TDD(테스트 주도 개발)에 기반한 검증 코드가 마련되어 있습니다.
*   명령어 하나로(예: `pytest tests/`) 데이터베이스 병목, 음악 큐 엣지 케이스, 경험치 정확도 등 40여 가지 이상의 상황극을 시뮬레이션해 봇 건강 상태를 진단할 수 있습니다. (`docs/TEST_GUIDE.md` 참조)

### 5. 데이터 컨테이너 (Data Directory)
이 폴더는 로컬 라즈베리파이 환경에서 구동/정리되며 깃허브에는 기본적으로 업로드되지 않습니다.
*   `bot_database.db`: 사용자 경험치 레벨, 음악 볼륨 등 핵심 데이터를 영구 보관하는 SQLite3 데이터베이스 파일.
*   `logs/`: 날짜별로 순환(Rotation) 생성되는 `system.log` 파일들이 보관됩니다.
