# Changelog

모든 주요 변경 사항과 아키텍처 패치 노트가 이 파일에 기록됩니다.

이 문서는 [Keep a Changelog](https://keepachangelog.com/ko/1.0.0/)의 국제 포맷팅 규칙을 따르며,
버전 넘버링은 [Semantic Versioning (유의적 버전)](https://semver.org/lang/ko/) 표준 규칙인 `vX.Y.Z` 방식을 엄수합니다.

---

## [v1.3.1] - 2026-06-21

### Added
- 사용자가 웹 브라우저를 모두 닫거나 퇴장한 후 5초 뒤 세션과 플레이리스트 데이터를 완전히 자동 파기하는 5초 유예식 세션 자동 소멸(Self-Destruct) 메커니즘을 추가했습니다.

### Changed
- 라즈베리파이 SD 카드 보호 및 성능 최적화를 위해 SQLite 데이터베이스 연결 시 WAL(Write-Ahead Logging) 저널 모드, NORMAL 동기화, MEMORY 임시 저장소, 2MB 페이지 캐시를 일괄 강제 활성화하도록 개선했습니다.

## [v1.3.0] - 2026-06-20

### Added
- 외부 브라우저용 실시간 유튜브 동시 시청(Watch Together) 시스템을 새롭게 구현했습니다.
- FastAPI 비동기 웹서버 및 WebSocket을 활용한 실시간 상태 동기화 중계 모듈(`cogs/watch_together/watch_server.py`)을 탑재했습니다.
- 디스코드 슬래시 명령어 `/시청`을 통한 UUID 기반 방 세션 개설 및 웰컴 링크 임베드 전송 기능을 구현했습니다.
- YouTube IFrame API와 웹소켓 양방향 통신을 조립한 모던 Glassmorphism UI 스타일의 전용 웹 플레이어 화면(`templates/player.html`)을 추가했습니다.

### Changed
- 로컬 윈도우/리눅스 개발 편의성을 극대화하기 위해 파이썬 가상환경(`.venv` 및 `bot_env`) 생성, 패키지 설치 및 실행 절차에 대한 가이드라인을 `README.md` 및 `project_context.md`에 공식 추가했습니다.

### Removed
- 데이터베이스 최적화의 후속 작업으로 `database_manager.py` 내에 임시로 놔두었던 레거시 생일 데이터 마이그레이션 및 birthdays 테이블 파기 로직 코드를 완전히 지우고 최적화했습니다.

## [v1.2.2] - 2026-06-07

### Refactored
- 데이터베이스 최적화를 위해 `birthdays` 테이블을 `users` 테이블로 단일화(병합)하는 자동 마이그레이션 로직을 도입했습니다.
- 과거 JSON 파일 시절에 사용되었으나 현재는 불필요해진 레거시 마이그레이션 함수(`migrate_json_to_db`)와 상수들을 일괄 제거하여 파일 용량을 크게 줄이고 시스템 무결성 테스트 항목을 현행화했습니다.

## [v1.2.0] - 2026-06-06

### Added
- 사용자의 생일을 기록하고, 매일 오전 9시 정각에 요약 채널을 통해 `@everyone` 멘션과 함께 생일을 축하해 주는 **생일 관리 시스템**을 추가했습니다.
- 데이터베이스에 `birthdays` 테이블을 새롭게 추가하였으며, 관리자 전용 커맨드(`/생일등록`, `/생일삭제`) 및 공개 커맨드(`/생일목록`)를 탑재했습니다.

### Changed
- 최고 관리자가 음성 채널에 있지 않은 상태로 노래를 강제로 추가할 경우, 봇의 인터페이스에 표기되는 요청자의 명의가 최고 관리자가 아닌 **봇(Bot)**으로 노출되도록 렌더링 방식을 변경하여 **자동 재생 연출**을 강화했습니다.

### Fixed
- 테스트 환경(`pytest`)에서 봇 구동 시 내부적으로 호출되는 `subprocess.run` 함수의 모의 객체(Mock) 반환값이 누락되어 테스트가 실패하던 버그를 수정하여 테스트 신뢰성을 복구했습니다.

## [v1.1.0] - 2026-04-19

### Changed
- 데이터베이스 덤프 백업 대상을 `main` 브랜치에서 분리하여, 단일 커밋 강제 덮어쓰기 방식으로 구동되는 `db-backup` 독립 브랜치 전용으로 전환했습니다. (저장소 용량 기하급수적 팽창 문제 완벽 해결 방어)

### Added
- 라즈베리파이 내부 오프라인 환경(`data/archives/`)에 최대 7일간 유지되는 로컬 롤백 타임머신을 구축했습니다.
- 시스템 완전 소실 후 첫 기동 시, 관리자의 개입 없이 원격 `db-backup` 브랜치에서 덤프 파일을 백그라운드로 즉각 다운로드(fetch)해 스스로 복구하는 **Zero-Touch Auto Recovery 시스템**을 탑재했습니다.

### Fixed
- 네트워크 통신 단절 시 봇이 빈(Empty) DB를 무단 생성한 뒤 빈 깡통을 깃허브에 밀어버려 백업본이 파괴되는 **Data Loss Loop 취약점**을 막기 위해, 통신 실패 시 `sys.exit()`으로 봇을 강제 셧다운 하도록 논리 구조를 개편했습니다.
- 생성된 덤프 파일 용량이 너무 작을 경우(1KB 미만) 깃허브 업로드를 원천 차단하는 이중 데이터 무결성 안전장치를 도입했습니다.
- 자동 덤프 임시 작업 경로를 고정 문자열(`/tmp/bot_db_backup`) 방식에서 운영체제가 보증하는 무작위성 폴더 발급(`mktemp -d`) 방식으로 변경하여 찌꺼기 파일에 의한 쓰기 충돌 버그 환경을 배제시켰습니다.

### Security
- 봇의 데이터 복구 동작 수행 중, 발생할 수 있는 메인 브랜치의 **Git Index(Staging 영역) 오염(Pollution)**사태로 인해 사용자의 암호화 정보가 엉뚱한 브랜치에 노출될 수 있는 치명적 엣지 케이스를 막기 위해, 파일 복원 방식 로직을 일차원적인 `git checkout`에서 `git show` 기반의 데이터 스트림 추출 방식으로 대폭 강화했습니다.
